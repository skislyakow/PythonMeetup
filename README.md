# PythonMeetup Bot

Telegram-бот для управления Python-митапами. Гости задают вопросы спикерам, организаторы управляют программой — всё через постоянное меню в Telegram и админку Django.

## Возможности

### 👤 Гость

| Кнопка | Описание |
|--------|----------|
| 📅 Программа | Список докладов, сгруппированный по датам, с отметкой текущего активного доклада |
| ❓ Задать вопрос | Отправить вопрос текущему спикеру (кнопка скрыта, если нет активного доклада) |
| ❤️ Поддержать проект | Донат (тестовый, произвольная сумма) |
| ℹ️ Справка | Роль-специфичная подсказка по кнопкам и командам |

### 🎤 Спикер

| Кнопка / Команда | Описание |
|------------------|----------|
| 📥 Мои вопросы (Спикер) | Просмотр вопросов с пагинацией (◀️ Предыдущий / Следующий ▶️) |
| ✍️ Ответить | Отправить ответ гость — гость получает уведомление |
| ⏭ Пропустить | Перейти к следующему вопросу |
| ℹ️ Справка | Подсказка по доступным действиям |

### 🔧 Организатор

| Кнопка / Команда | Описание |
|------------------|----------|
| `/admin` | Панель организатора с полной клавиатурой |
| ✏️ Создать доклад | Пошаговое создание: название → дата → время начала → время конца → спикер → подтверждение |
| 📅 Управление событиями | Список докладов с кнопками: ✅ активировать, ⏹ завершить, ✏️ редактировать, 🗑️ удалить |
| ✏️ Редактирование | Выбор поля (название, спикер, дата, начало, конец) → новое значение → подтверждение |
| 📢 Уведомления | Автоуведомление всех участников при старте / завершении доклада |
| `/broadcast` | Рассылка произвольного сообщения всем пользователям |
| 🔄 Автороли | Автоназначение роли спикера при создании / активации доклада |
| ℹ️ Справка | Подсказка по доступным действиям |

### 🖥️ Django Admin

- Управление пользователями: смена роли (`guest` / `speaker` / `organizer`)
- Управление событиями: флаг `is_active`, просмотр всех докладов
- Просмотр вопросов и ответов

## Технологии

| Компонент | Технология |
|-----------|------------|
| Backend | Python 3.12+, Django 5.x |
| Bot Framework | python-telegram-bot v22 (ConversationHandler, CallbackQueryHandler) |
| База данных | SQLite (Django ORM) |
| Конфигурация | python-dotenv (`.env`) |
| Разметка | HTML (parse_mode) |

## Архитектура

```
Update → Dispatcher → Handler (guest/speaker/organizer) → Django ORM → SQLite
```

- **handlers/** — обработчики команд и callback-запросов, разделённые по ролям
- **services/** — бизнес-логика (клавиатуры, авторизация, запросы к БД)
- **models/** — Django-модели (TelegramUser, Event, Question)
- **ConversationHandler** используется для многошаговых сценариев (создание доклада, редактирование, донат)

Общение с Telegram — через `python-telegram-bot`, хранение — через Django ORM. Это позволяет одновременно управлять данными через админку Django и через бота.

## Быстрый старт

```bash
git clone <repo-url>
cd PythonMeetup
python -m venv venv
# Windows: venv\Scripts\activate
# Linux/macOS: source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
```

Заполнить `.env`:
- `BOT_TOKEN` — токен от [@BotFather](https://t.me/BotFather)
- `ORGANIZER_IDS` — свой Telegram ID (узнать: @userinfobot)

```bash
python manage.py migrate
python -m bot.seed       # тестовые данные
python -m bot.main       # запустить бота
```

## Админка Django

```bash
python manage.py createsuperuser
python manage.py runserver
```

Открыть `http://127.0.0.1:8000/admin/`.

```
│   ├── handlers/          # guest.py, organizer.py, speaker.py, donates.py, help.py

│   └── seed.py            # тестовые данные

├── manage.py
└── requirements.txt
```

## Разработка

Проект открыт для улучшений. Создавайте своего тестового бота через [@BotFather](https://t.me/BotFather) и вписывайте свой токен в `.env`. Для организаторских функций укажите свой Telegram ID в `ORGANIZER_IDS`.
